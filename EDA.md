# 📡 통신사 고객 이탈 예측 및 분석 프로젝트

## 1. 프로젝트 주제
본 프로젝트의 주제는 **통신사 고객 이탈(Churn)을 데이터 분석과 머신러닝을 통해 예측하고, 기업이 실제로 활용 가능한 의사결정 근거와 실행 전략을 도출하는 것**입니다.

단순히 모델 정확도를 높이는 것이 아니라,

> **"왜 고객이 이탈하는가?" → "어떤 고객을 언제 관리해야 하는가?" → "이탈을 줄이면 매출은 얼마나 회복되는가?"**

라는 기업 관점의 질문에 답하는 것을 목표로 합니다.

---

## 2. 문제 정의 (Business Problem)
통신 산업은 신규 고객 확보 비용(CAC)이 기존 고객 유지 비용보다 매우 높습니다.  
따라서 고객 이탈은 단순한 고객 수 감소가 아니라 **직접적인 매출 손실**로 이어집니다.

본 프로젝트에서는 다음 문제를 정의했습니다.
- 고객 이탈은 무작위로 발생하는가?
- 이탈 직전에 반복적으로 나타나는 행동 패턴은 무엇인가?
- 사전 개입 시, 기업은 **얼마나 큰 재무적 효과**를 얻을 수 있는가?

---

## 3. 데이터 개요
- 데이터 유형: 통신사 고객 사용 이력 데이터
- 분석 단위: 고객 단위(Customer-level)
- Target 변수
  - `churn` : 고객 이탈 여부 (1 = 이탈, 0 = 유지)

### 주요 Feature 그룹
- 고객 행동
  - `number_customer_service_calls`
- 요금제 정보
  - `international_plan`, `voice_mail_plan`
- 사용량
  - `total_day / eve / night / intl_minutes`
- 요금
  - `total_day / eve / night / intl_charge`

📌 **분석 가정**  
- 각 Charge 변수의 합을 고객의 월 매출(`total_revenue`)로 가정하여 재무 분석에 활용

---

## 4. EDA 기반 핵심 분석

### 4.1 고객 이탈 분포
- 전체 고객 중 약 **14% 내외가 이탈 고객**
- 이탈 비율은 낮지만, 이탈 고객이 발생시키는 **매출 손실 비중은 매우 큼**

➡️ *이탈률 자체보다 재무적 영향 분석이 중요*

---

### 4.2 고객센터 문의 횟수 분석 (Key Risk Signal)
- 고객센터 문의 횟수가 증가할수록 이탈률이 계단식으로 상승
- **3~4회 이상 문의 고객군**에서 평균 대비 급격한 이탈률 증가

**해석**  
- 반복적인 고객센터 접촉은 일시적 불만이 아닌 **이탈 직전 누적 신호**

---

### 4.3 국제전화 사용 패턴 분석
- 국제전화 요금제 가입자 및 고사용자군에서 평균 대비 높은 이탈률
- 가격 민감도가 높고 경쟁사 요금 비교 가능성 큼

---

### 4.4 요금·사용량 변수 상관관계
- `minutes` ↔ `charge` 변수 간 매우 강한 양의 상관관계
- 다중공선성 존재 → 트리 기반 모델(CatBoost) 적합

---

## 5. 분석 결론 (Insight Summary)
1. 고객 이탈은 **무작위 현상이 아니라 반복되는 패턴을 가진 사건**이다.

"""
고객 행동 패턴: (1 -> 5 순서대로 고객의 행동이 발생) 
1. 요금/서비스 불만 발생 
2. 고객센터 반복 문의
3. 불만 해결 실패 또는 기대 미달
4. 대안 탐색(타사 비교)
5. 이탈 
"""

2. 고객센터 문의 횟수와 국제전화 사용 여부는 가장 강력한 이탈 신호다.
3. 이탈은 곧 매출 손실이며, **관리 대상 고객을 선별하면 충분히 줄일 수 있다.**

## 6. 시뮬레이터 설계 및 의미
본 프로젝트는 분석 결과를 실제 의사결정으로 연결하기 위해 **전략 시뮬레이터**를 설계했습니다.

### 시뮬레이터 개념
- 대상 고객군
  - 고객센터 문의 3회 이상 고객
  - 국제전화 요금제 가입자
- 전략

  - 케어 프로그램
  """
  1. 우선 상담 케어 (전담 상담원 배정, 이전 문의 이력 자동 조회)
  - 이유: 고객센터 반복 접촉은 이탈 직전 신호
  2. 한시적 비용 보상 (국제전화 요금 일부 캐시백, 다음달 요금 할인)
  - 포인트: 전 고객 할인 X -> 위험 고객 한정 + 단기 제공
  3. 문제 해결 중심 케어 (단순 사과 X -> 네트워크, 요금, 서비스 오류 유형별 해결 시나리오 구축)
  - 핵심: 고객은 할인보다 “이제 문제 안 생기겠구나” 를 원함
  """

  - 요금제 개편
  """
  1. 국제전화 요금제 세분화 (고정형 요금제 -> 사용량 기반 요금제)
  - 근거: 국제전화 고사용 고객군에서 이탈률 높음 → 가격 충격 완화 필요

  2. 불만 유발 요금 구간 제거 (특정 요금 구간에서 사용량 대비 요금 급증 현상 존재 -> 해당 구간 완만화 또는 제거)
  - 효과: 고객센터 문의 감소, 장기 이탈률 감소 

  3. 번들, 옵션형 요금제 (국제전화 + 데이터 옵션 결합, 단기 해외 이용 고객용 패스)
- 효과: 
| 구분    | 단기 할인 | 번들 요금제 |
| ----- | ----- | ------ |
| 효과    | 일시적   | 지속적    |
| 고객 인식 | 보상    | 구조 개선  |
| 이탈 재발 | 높음    | 낮음     |
| 매출 예측 | 불안정   | 안정적    |

  """

### 시뮬레이션 목적
- "전략을 실행하면 무엇이 달라지는가?"를 수치로 제시
- 모델 예측이 아닌 **경영 시나리오 기반 효과 추정**

## 7. 기대 효과 분석 (Expected Impact)

### 이탈률 관점
- AS-IS: 약 **14%**
- TO-BE: 약 **8~9% 수준까지 감소 가능**

### 재무 관점
- 이탈 감소는 곧 월 매출 손실 감소
- 연간 환산 시 **유의미한 매출 회복 효과 발생**

➡️ *이탈률 개선은 곧 비용 절감이 아닌 매출 증대*

## 9. 최종 결론 (Executive Conclusion)

> 고객 이탈은 우연이 아니라 **관리 가능한 경영 변수**입니다.

본 분석은 다음을 보여줍니다.
- 이탈 직전 고객은 사전에 식별 가능
- 제한된 자원으로도 핵심 고객군에 집중하면 효과 극대화 가능
- 데이터 기반 개입은 **이탈률 감소 → 매출 회복**으로 직결됨
